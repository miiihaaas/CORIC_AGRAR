# Story 8.7: Blog CRUD Admin sa WYSIWYG

Status: ready-for-dev

## Story

As a **Marijana** (sadržaj-urednik / Editor RBAC rola),
I want **da pišem i uređujem blog objave kroz pun admin CRUD sa rich-text (WYSIWYG) editorom za telo objave i multi-locale tabovima (sr/hu/en)**,
so that **mogu da formatiram tekst (naslovi, liste, podebljano, linkovi) bez dev intervencije, a da pri tome rendered HTML NIKAD ne postane stored-XSS vektor**.

## Opis

Sedma Epic 8 story i **ČETVRTA (i POSLEDNJA per-content-model) CRUD admin** u nizu 8.4 (Brand) → 8.5 (Category/Subcategory) → 8.6 (Product) → **8.7 (Blog Post/Category/Tag)**. Nadograđuje POSTOJEĆI `apps/blog/admin.py` STUB (5-1 SM-D8: `TranslationAdmin` + `list_display`/`list_filter`/`search_fields=("title_sr",)` + `prepopulated_fields` slug + `SeoMetaInline` + `view_on_site=False`) u pun multi-locale CRUD obrazac koji 8.4/8.5/8.6 utvrdili.

**NOVI THRUST ove story** (jedinstven u Epic 8, NIJE u prethodne 3 CRUD admin story): **WYSIWYG editor za `Post.body` + render-time `nh3` sanitizacija** koja zatvara stored-XSS granicu. Ovaj rich-text + sanitizacija posao je EKSPLICITNO odlagan kroz ceo projekat na „Epic 8.7": 5-1 SM-D10 (body=plain TextField, editor=8.7), 5-3 SM-D1 (KEEP `|linebreaks`, DEFER rich-text/sanitizacija na 8.7), 7-1/7-4 (legal body plain `|linebreaks`, WYSIWYG defer 8.7), 7-5 (nh3 sanitizer izgrađen u `apps.core` EKSPLICITNO za forward-reuse 8.7 blog), 8.6 SM-D2/OQ-1 (Product description WYSIWYG defer, „briga deli sa 8.7"). **8.7 je tačka naplate svih tih deferral-a.**

Blog model foundation (5-1): `Post(SluggedModel + TimestampedModel)` sa `title`/`perex`/`body` translatable (_sr/hu/en), `main_image` ImageField (blank/null), `category` FK SET_NULL, `tags` M2M, `author` FK `settings.AUTH_USER_MODEL` SET_NULL, `status` TextChoices `DRAFT`/`PUBLISHED` (JEDAN status field — NE products-stil dual is_published+status; 5-1 SM-D2), `published_at` DateTimeField. `Post.objects` default PRVI + `Post.published` (PublishedManager filtrira `status='published' AND published_at__lte=now`). `Category(name/description translatable)` + `Tag(name translatable)` — slug auto-gen iz `name`/`title` kroz `slugify_ascii` u `save()`/`full_clean()`. `body` ima model-nivo `MaxLengthValidator(50000)` belt-and-suspenders hard cap (5-1 Code Review M).

**SCOPE GRANICA (SM-D1):** 8.7 = SAMO Blog admin (`Post` + `Category` + `Tag`) u `apps/blog/admin.py`. Brand/Series (8.4 DONE), Category/Subcategory brands (8.5 DONE), Product+6 inline (8.6 DONE) su NETAKNUTI. 8.8 (statičke strane Page) i 8.9 (SiteSettings/Navigation) su KASNIJE.

## Acceptance Criteria

1. **AC1 — TranslationAdmin multi-locale za Post/Category/Tag.** `PostAdmin`, `CategoryAdmin`, `TagAdmin` su `TranslationAdmin` (kao danas u stub-u) — modeltranslation AUTO-renderuje sr/hu/en tabove za `title`/`perex`/`body` (Post), `name`/`description` (Category), `name` (Tag). `fieldsets` (gde postoje) koriste BAZNA imena polja (`title`/`perex`/`body`/`name`/`description`) — TranslationAdmin auto-ekspanduje u per-locale; NE enumerisati `_sr/_hu/_en` ručno (G-1). `PostAdmin` MRO je `SeoWarningAdminMixin, TranslationAdmin` (mixin PRVI — G-2).

2. **AC2 — WYSIWYG editor za `Post.body`.** `Post.body` (i samo body; NE perex, NE title) renderuje se u admin change/add formi kao rich-text WYSIWYG editor (toolbar: paragrafi, h2/h3/h4 headings, bold/italic, link, unordered/ordered lista). Editor je vezan na SVA tri locale tab-a (`body_sr`/`body_hu`/`body_en`) — svaki body tekstarea dobija WYSIWYG. Editor radi BEZ build-pipeline-a (project-context: „no build pipeline — JS se servira direktno"), BEZ jQuery (project-context JS rule: „Vanilla JS preferred ... NIKAD jQuery"), i NE menja `Post.body` model field (ostaje plain `TextField` koji čuva HTML string — SM-D2). Editor izbor sledi deterministički tiebreaker SM-D2 (Grana A 0-dep no-jQuery, ILI Grana B default vanilla-JS minimalni toolbar bold/italic/h2-h4/link/ul/ol iz `static/js/`) — toolbar je BOUNDED na taj skup tagova (svi unutar nh3 allowlist-a), NIKAD from-scratch full editor. **PROGRESSIVE ENHANCEMENT:** WYSIWYG je enhancement IZNAD običnog `<textarea>`; podležno polje ostaje plain `Textarea` → ako JS otkaže, admin i dalje radi (Marijana piše u textarea, save prolazi). Add-view i change-view oba renderuju editor → 200 (NE admin.E* / NE JS-crash render fail).

3. **AC3 — Render-time `nh3` sanitizacija `Post.body` (STORED-XSS GRANICA).** Javni blog detail (`templates/blog/post_detail.html:45`) renderuje `Post.body` kroz REUSE postojećeg `apps.core` 7-5 sanitizera (`sanitize_legal_html` + `{{ value|legal_html }}` filter) ILI kroz blog-specifičan ekvivalent koji deli isti `nh3` allowlist mehanizam — `nh3.clean(...)` na svakom renderu, pa `mark_safe(...)` SAMO POSLE sanitizacije. `Post.body` se NIKAD ne renderuje kroz `|safe` bez prethodne `nh3` sanitizacije, NIKAD kroz `|linebreaks` (koji ESCAPE-uje → double-escape sirovog HTML-a; 5-3 post_detail.html:44 komentar). Adversarni payload (`<script>alert(1)</script>`, `<img src=x onerror=...>`, `javascript:` href, `onclick` atribut, `<iframe>`) u `Post.body` → STRIP/neutralizacija u rendered HTML-u (nh3 STRIPUJE node van allowlist-a, NE escape-uje — SM-D7 7-5 presedan). `<h1>` se NE renderuje iz body-ja (rezervisan za `{% translated_field post 'title' %}` template H1 — 7-5 G-8). Render-time je PRIMARNA XSS granica — NIKAD se ne veruje stored vrednosti (7-5 SM-D3).

4. **AC4 — `main_image` upload MIME + decompression-bomb guard (REUSE media_pipeline).** `Post.main_image` (ImageField, blank=True/null=True) upload validira se kroz blessed `apps.media_pipeline.utils.validate_image_mime` (MIME signature via python-magic + Pillow `verify()` + size cap + `MAX_IMAGE_PIXELS=50M` decompression-bomb guard) — NIKAD reimplementirana inline validacija (8.4 M2 / 8.6 SM-D6 HARD lekcija). `main_image` je override-ovan u plain `forms.FileField(required=False)` jer Django ImageField form-field pokreće SVOJ Pillow `to_python()` PRE `clean_main_image` → pregazi srpsku media_pipeline poruku (8.4 lekcija d / 8.6 G-14). `clean_main_image` blank-skip-uje prazan upload (main_image blank=True), inače delegira na helper. 81M-px PNG bomba ispod size-cap-a (8.6 Security runtime presedan) → ODBIJENA pixel-guardom. Lokalne konstante `MAX_IMAGE_UPLOAD_SIZE = 5 * 1024 * 1024` + `ALLOWED_IMAGE_MIME_TYPES = ("image/jpeg", "image/png", "image/webp")` (SVG izostavljen — XSS vektor; mirror 8.4/8.6). **NAPOMENA: `validate_image_mime` DEFAULT `max_size_bytes` je 10MB (`MAX_UPLOAD_SIZE_BYTES`, media_pipeline/utils.py:46,52) — MORA se EKSPLICITNO override-ovati prosleđivanjem `max_size_bytes=MAX_IMAGE_UPLOAD_SIZE` (5MB) u pozivu, inače bi blog koristio 10MB umesto 5MB cap-a (mirror products/admin.py:68).**

5. **AC5 — Publish-gate (graceful draft revert, NIKAD 500).** Pri pokušaju objave (`status == Post.Status.PUBLISHED`) admin validira da sr verzija ima: `title_sr` (modeltranslation ga VEĆ promoviše u required — AC9) + `body_sr` neprazan + `main_image` priložen + `category` izabran (epics:1151: „validacija: sr verzija ima title + body + main_image + category"). Na neuspeh: graceful `messages.error` + revert `status` na `Post.Status.DRAFT` kroz direktan `QuerySet.update()` bypass (NE `instance.save(update_fields=...)` — `Post.save()` bezuslovno poziva `full_clean()` i ignoriše `update_fields`, models.py:201-205 → latentni 500 footgun; 8.6 B1 lekcija). NIKAD `raise` iz `save_related`/`clean` na publish-gate (raise → HTTP 500; 8.6 G-6). Draft save (`status == DRAFT`) se NIKAD ne gate-uje. **NAPOMENA reconcile:** gate proverava count/prisustvo inline-nezavisnih polja (title_sr/body_sr/main_image/category su SVA na Post-u samom, NEMA inline-a kao kod 8.6 galerije/spec) → gate može biti u `clean()`/`save_related` (Dev bira; pošto nema inline count-a, gate je jednostavniji od 8.6 — vidi SM-D5).

6. **AC6 — `published_at` auto-set pri publish-u ako prazno.** Kad se objava prebaci u `status=PUBLISHED` a `published_at` je prazno (`None`), admin postavlja `published_at = timezone.now()` (epics:1152). NIKAD `datetime.now()` — uvek `django.utils.timezone.now()` (project-context anti-pattern naive datetime). Ako `published_at` već ima vrednost (admin ručno postavio budući/prošli datum), NE pregazi je. Reconcile sa PublishedManager: `Post.published` filtrira `published_at__lte=now` → objava sa budućim `published_at` je „zakazana" (validan scenario, ne menjati).

7. **AC7 — `Tag` slobodno dodavanje + `Category` FK dropdown na PostAdmin.** `Post.category` (FK) renderuje se kao default FK `<select>` widget. `Post.tags` (M2M) renderuje se sa „dodaj novi tag" affordance — kroz `filter_horizontal`/`filter_vertical` (M2M dual-list) ILI raw-id sa related-widget `+` add-popup (epics:1148: „inline-ove za Tag (slobodno dodavanje)"). Marijana može dodati novi Tag inline (kroz related-object `+` popup) BEZ napuštanja Post forme. TagAdmin i CategoryAdmin su zasebno registrovani (changelist/add/change → 200) za nezavisan CRUD.

8. **AC8 — SeoMetaInline + SeoWarningAdminMixin regression OČUVAN (HARD lock).** `PostAdmin.inlines` MORA sadržati `SeoMetaInline` (6.1 regression — G-8) i `PostAdmin` MORA ostati `SeoWarningAdminMixin` subclass (soft meta-warning mehanizam). `PostAdmin` MORA ostati `TranslationAdmin` instanca. Postojeći regression-lock testovi u `apps/seo/tests/test_seometa_admin_inline.py` (`test_seometa_inline_registered_on_receiving_admins` koji uključuje `Post`, `test_postadmin_stays_translationadmin_and_view_on_site_false`, `test_post_add_view_renders_inline_200`) MORAJU OSTATI ZELENI. **KRITIČNO (re-enable odluka):** `view_on_site` — 5-1 IMP-1/BL-5 postavio `view_on_site=False` jer `blog:detail` nije postojao; 5-3 JE registrovao `blog:detail` (`/sr/blog/<slug>/`). Story MORA doneti odluku (SM-D8): re-enable „View on site" (ukloni `view_on_site=False`) jer URL sada postoji, ILI zadržati False. Ako re-enable → `test_postadmin_stays_translationadmin_and_view_on_site_false` se MORA AŽURIRATI (taj test je test-ownership na 8.7; 5-x lekcija „test-ownership kad story menja tuđi markup/ugovor").

9. **AC9 — `title_sr` bezuslovno obavezan (modeltranslation required-promotion — NE relaksiraj).** `Post.title` je `blank=False` + sr je default lang → modeltranslation promoviše `title_sr` u required form polje. Story NE relaksira `title_sr.required` (8.4 M1 BUG / 8.6 AC9 HARD lekcija). Prazan `title_sr` → graceful FORM greška (200), NIKAD model `full_clean()` escape (400/500). `body`/`perex` su `blank=True` na modelu → NISU obavezni na form-sloju (draft može imati prazan body; publish-gate AC5 traži `body_sr` tek pri objavi). Bazno `title` polje (kad raw ModelForm vidi i `title` i `title_sr`) može biti relaksirano kroz core helper (SM-D6) — `title_sr` OSTAJE required.

10. **AC10 — `search_fields` realna kolona + RBAC verify (NE re-grant).** `search_fields` koristi REALNE DB kolone (`title_sr` za Post, `name_sr` za Category/Tag) — NE virtuelni translated `title`/`name` (runtime FieldError na changelist search; G-1). RBAC: 8.2 `EDITOR_CONTENT_MODELS` VEĆ sadrži `("blog", "post")` + `("blog", "category")` + `("blog", "tag")` → Editor ima add/change/delete/view. 8.7 NE re-grant-uje RBAC (samo verifikuje da Editor → GET `admin:blog_post_changelist` → 200, postojeći `apps/accounts/tests/test_rbac.py:test_ac9_editor_can_access_blog_post_changelist_200` lock OSTAJE zelen).

11. **AC11 — 0-or-N migracija accounting (admin-only = 0 migracija očekivano).** Ako 8.7 NE menja nijedan model (čista admin promena), `makemigrations blog --check` = „No changes" (0 migracija). Ako WYSIWYG izbor zahteva novi dep koji dodaje INSTALLED_APPS app sa migracijama (npr. `django-tinymce` NEMA modele → 0 blog migracija; ako se ipak doda neki app, te migracije su VLASTITE tog app-a, NE blog), eksplicitno dokumentovati. Očekivanje (SM-D2 default): 0 blog migracija. Manual review svake migracije ako se ipak pojavi (project-context migrations discipline).

12. **AC12 — `manage.py check` čist + ŠIRI suite regression zelen.** `manage.py check` (system checks uključujući admin.E*) = 0 serious errors (prepopulated_fields/search_fields + modeltranslation + WYSIWYG widget ne smeju baciti admin.E030/E116/FieldError; 5-1 BL-2 guard).
