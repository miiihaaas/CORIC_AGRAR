# Story 6.1: SeoMeta Model + Per-page Admin

Status: ready-for-dev

## Opis

As a **admin (koji unosi SEO meta za strane, proizvode, brendove i blog objave)**,

I want **NOVI `apps/seo/` app sa `SeoMeta` modelom (Generic FK na bilo koji objekat, translatable `meta_title`/`meta_description`, `og_image`, `exclude_from_sitemap` flag), modeltranslation registracijom (`_sr/_hu/_en`), generic admin inline-om na Product/Brand/Post admin-u, i `{% seo_meta object %}` template tag-om koji renderuje meta u `<head>` sa default fallback-om**,

so that **kontrolišem kako se svaka strana/proizvod/brend/objava prikazuje u Google rezultatima i social preview-ima — sa razumnim default-om (display title + naziv firme) kad SeoMeta nije ručno unesen**.

Ovo je **PRVA story Epic 6 (SEO & Discoverability)** i čista **GENERIC-META FOUNDATION**: TEMELJ na koji se 6.2 (sitemap `exclude_from_sitemap` + hreflang) i 6.3 (full Open Graph + Twitter Card + robots.txt) oslanjaju. **U OVOJ STORY UVODIMO PRVU `GenericForeignKey` u projektu** (grep potvrdio — nijedan postojeći model je ne koristi). 6.1 polaže model + translation + migraciju + generic admin inline + template tag (title/desc/canonical/og:image) + default fallback + soft warning.

### IN SCOPE (šta ova story isporučuje)

1. **NOVI `apps/seo/` app** (SM-D1) — `SeoConfig` (`name="apps.seo"` + `verbose_name=_("SEO")`) + registracija u `INSTALLED_APPS` (mirror apps/blog 5-1 scaffolding). Konzument modeltranslation paketa + `django.contrib.contenttypes` (već prisutan, base.py:37).
2. **`SeoMeta` model** (TimestampedModel) — `content_type` (FK→`contenttypes.ContentType`, on_delete=CASCADE), `object_id` (PositiveIntegerField, db_index), `content_object` (GenericForeignKey), `meta_title` (CharField translatable), `meta_description` (TextField translatable), `og_image` (ImageField nullable upload_to="seo/og/"), `exclude_from_sitemap` (BooleanField default=False). `UniqueConstraint(content_type, object_id)` (SM-D4) = jedan SeoMeta po objektu + služi kao GFK lookup indeks.
3. **`apps/seo/translation.py`** (SM-D9) — `@register(SeoMeta)` `fields=("meta_title", "meta_description")` (mirror blog 5-1). Generiše `meta_title_sr/_hu/_en` + `meta_description_sr/_hu/_en` → DB kolone kroz makemigrations.
4. **Schema migracija** `apps/seo/migrations/0001_initial.py` (CreateModel SeoMeta + `_sr/_hu/_en` kolone + GFK fields + UniqueConstraint + index — manuelno reviewovana, SM-D4). **NEMA data seed.**
5. **`apps/seo/admin.py:SeoMetaInline`** (SM-D3) — GENERIC inline (`GenericStackedInline` baza + modeltranslation prevod meta polja kroz mixin — vidi Gotcha SEO1-2) žičan kao `inlines=[SeoMetaInline]` na admin-e: Product, Brand, Series, Subcategory, Post (+ Page/SiteSettings — vidi OQ-1). **Soft warning** (SM-D5) kad `meta_title>60` ili `meta_description>160` — NON-BLOCKING (admin `message_user` WARNING, NE `add_error` koji bi blokirao save).
6. **`apps/seo/templatetags/seo_meta.py:{% seo_meta object %}`** (takes_context=True; SM-D2/D7) — render `<title>`/`<meta name="description">`/`<link rel="canonical">`/`<meta property="og:image">` u `<head>`. **NO-DUPLICATE-<title> integracija** kroz dva komplementarna tag-a (SM-D2): `{% seo_title object %}` (vraća STRING vrednost za `{% block title %}`) + `{% seo_head object %}` (emituje canonical+og:image u `extra_head`). Default fallback (SM-D1-FALLBACK) kad nema SeoMeta: `meta_title` = display title + " | " + company_name; `description` = `perex → description → (sopstveni) slogan → display_title(name/title) → SiteSettings.slogan → ""` (CRIT-1 — `display_title` rung PRE site-wide slogana čuva blog `perex|default:post.title` semantiku).
7. **1 migracija** (`apps/seo/0001_initial`). **NEMA novog dep-a** (contenttypes built-in base.py:37; modeltranslation već instaliran; ImageField → Pillow prisutan kroz sorl-thumbnail pyproject.toml:21).

### OUT OF SCOPE (eksplicitno — granice)

- **Sitemap (`apps/seo/sitemaps.py` + `config/urls.py` `path("sitemap.xml", ...)` + hreflang `alternates`)** = **Story 6.2** (epics.md:927-939). 6.1 SAMO uvodi `exclude_from_sitemap` polje koje 6.2 KONZUMIRA (`exclude_from_sitemap=True` → izostavi iz sitemap-a). 6.1 NE kreira sitemap klase ni URL.
- **Full Open Graph card (`og:title`/`og:description`/`og:type`/`og:url`) + Twitter Card meta + robots.txt (`apps/seo/views.py:robots_txt`)** = **Story 6.3** (epics.md:941-952). 6.1 renderuje SAMO `og:image` (epics.md:923 EKSPLICITNO navodi `<meta property="og:image">` u 6.1; pun OG/twitter set je 6.3). 6.1 `og_image` polje + render `og:image` su FORWARD-SUPPORT za 6.3 (koji dodaje ostale og:* tagove + `static/img/og-default.jpg` fallback + twitter:card — epics.md:949/952). 6.1 NE kreira robots.txt ni views.py.
- **Redirect Manager (301) (`apps/seo/models.py:Redirect` + `apps/seo/middleware.py:RedirectMiddleware`)** = **Story 6.4** (epics.md:954-964). 6.1 NE uvodi Redirect model ni middleware.
- **Lokalizovani slug-ovi / hreflang HTML tagovi** = **Story 6.6** (epics.md). 6.1 canonical je locale-prefiksovan kroz `i18n_patterns` (`reverse`/`get_absolute_url` daje aktivni-locale prefiks), ALI NE emituje `<link rel="alternate" hreflang>` (to je 6.2 sitemap + 6.6 HTML scope). 6.1 canonical = JEDAN aktivni-locale URL.
- **Pun SeoMeta CRUD admin polish (standalone changelist, bulk edit, OG image preview/crop, char-counter JS widget)** = **Epic 8 polish** (epics.md:922 EKSPLICITNO „(Epic 8 polish)"). 6.1 daje inline + soft warning; richer admin UX (live char counter, image crop, standalone SeoMeta menadžment strana) je Epic 8.
- **og:image dimenzije/aspect-ratio validacija (1200×630 OG preporuka)** — NE u 6.1 (OQ-3 deferred). 6.1 `og_image` je plain ImageField (Pillow validira da je slika); dimenzije/recommended-size validacija je Epic 8 polish ili 6.3.
- **Defensive validacija za nemoguće slučajeve** (project-context.md:358) — NE. `clean()` SAMO ako postoji stvaran invariant. Soft warning (>60/>160) je ADVISORY (NE-blocking) — NIJE `clean()` ValidationError (SM-D5).
- **GenericRelation reverse accessor na svakom receiving modelu (`seo_meta = GenericRelation(SeoMeta)`)** — NE u 6.1 (OQ-4). Tag radi forward lookup (`SeoMeta.objects.filter(content_type=..., object_id=...)`) bez dodavanja `GenericRelation` polja na 5 postojećih modela (izbegava 5 dodatnih migracija + dira 5 app-ova). `GenericRelation` (za cascade-delete + reverse query optimizaciju) je OPCIONI forward-optimization (6.2/Epic 8) ako se pokaže potreba.
