# Story 9.1: Production Docker Compose + Nginx Config

Status: review

## Story

As a **dev**,
I want **production Docker Compose stack sa Nginx reverse proxy ispred Gunicorn-a (security-hardened), koji se di≈æe lokalno za smoke test**,
so that **mogu da deploy-ujem na Hetzner CX32 sa istim, ponovljivim, security-hardened config-om (HTTPS, security headeri, direktan static serving) ‚Äî bez ruƒçnog server tuning-a po deploy-u**.

## Opis

**PRVA Epic 9 (Go-Live Readiness) story** ‚Äî uvodi production deployment artefakte. Ovo je **INFRASTRUKTURNA/DEPLOYMENT story**, NE Django app feature: veƒáina deliverable-a su **CONFIG fajlovi** (`compose/production.yml`, `compose/nginx/nginx.conf`, `compose/nginx/Dockerfile`, prod entrypoint/start skripta, Gunicorn config) + jedna nova runtime dep (`gunicorn`), a NE Python aplikativni kod. Testovi su **INFRA-VERIFY** (`docker compose config` lint + `nginx.conf` grep-asercije + prod-settings import check + opciono `curl -I` header smoke), **NE browser E2E** (`needs_e2e=false` ‚Äî `test-design-system.md`).

Nadograƒëuje POSTOJEƒÜU lokalnu Docker infrastrukturu (1-3): `production.yml` ogledava strukturu `compose/local.yml` (django + db servisi, healthcheck-ovi, named volume-i), reuse-uje `compose/django/Dockerfile` (2-stage uv build ‚Äî prod dodaje `--no-dev`), i EXTENDS veƒá-postojeƒái `config/settings/production.py` (1-2 ‚Äî DEBUG=False, SECURE_SSL_REDIRECT, HSTS, SECURE_PROXY_SSL_HEADER, Whitenoise manifest, X_FRAME_OPTIONS=DENY, SECURE_REFERRER_POLICY). Request flow (architecture:750‚Äì757): **Request ‚Üí Nginx (HTTPS termination, static fallback) ‚Üí Gunicorn (WSGI) ‚Üí Django**.

## Acceptance Criteria

1. **AC1 ‚Äî `compose/production.yml` defini≈°e prod servise (postgres + django/Gunicorn + nginx + glitchtip PLACEHOLDER).** Servisi: (a) `postgres` (postgres:16-alpine, healthcheck `pg_isready`, named volume za data, BEZ host port expose, credentials kroz env/secrets ‚Äî NIKAD `coric:coric` hardkodovan kao u local.yml); (b) `django` (build iz `compose/django/Dockerfile` sa prod `--no-dev` putom, `DJANGO_SETTINGS_MODULE=config.settings.production`, pokreƒáe **Gunicorn** NE runserver, `depends_on: postgres service_healthy`, BEZ host port expose ‚Äî interno :8000 samo); (c) `nginx` (build iz `compose/nginx/Dockerfile`, exposes 80 [i 443 za prod TLS], `depends_on: django`, mount static volume read-only); (d) `glitchtip` **PLACEHOLDER/zakomentarisan** servis sa `# Epic 9.3` markerom (epics:1190 ‚Äî NE realno wiring). `docker compose -f compose/production.yml config` izlazi 0 (validan YAML/Compose schema). Named volume-i: `postgres_data` (prod-scoped ime) + `staticfiles` (deljen django‚Üînginx).

2. **AC2 ‚Äî `compose/nginx/nginx.conf`: reverse proxy + gzip + direktan static + 3 security headera.** `nginx.conf` sadr≈æi: (a) `upstream`/`proxy_pass` ka `django:8000` (Docker DNS ime servisa) za SVE app-requeste; (b) `proxy_set_header X-Forwarded-Proto $scheme` (+ `Host`/`X-Real-IP`/`X-Forwarded-For`) ‚Äî KLJUƒåNO da `SECURE_PROXY_SSL_HEADER` u production.py radi; (c) `location /static/ {...}` koji servira DIREKTNO iz deljenog static volume-a (NE proxy na Django); (d) `location /media/ {...}` direktan (media volume); (e) `gzip on` (+ relevantni `gzip_types`); (f) tri security headera preko `add_header`: `X-Frame-Options "DENY"`, `X-Content-Type-Options "nosniff"`, `Referrer-Policy "same-origin"` (usagla≈°eno sa production.py vrednostima). Verifikacija: grep-asercije nad `nginx.conf` (proxy_pass django:8000, X-Forwarded-Proto, location /static/, gzip on, sva 3 headera) + (gde harness mo≈æe) `nginx -t` sintaksa.

3. **AC3 ‚Äî `docker compose -f compose/production.yml config` validira I `up` di≈æe stack lokalno za smoke sa REALNIM 200 round-trip-om.** `config` izlazi 0 sa oƒçekivanim servisima/kljuƒçevima (postgres, django, nginx + named volumes + `DJANGO_SETTINGS_MODULE=config.settings.production` u django env). `up` (lokalno, bez realnog cert-a) di≈æe postgres (healthy) ‚Üí django/Gunicorn (slu≈°a :8000) ‚Üí nginx (slu≈°a :80) i HTTP `GET /` (app-endpoint koji vraƒáa sadr≈æaj, npr. `/sr/` ili `/healthz`) kroz Nginx ‚Üí Django vraƒáa **HTTP 200** (NE 502 bad gateway, NE 301). **‚õî 301 NIJE prolaz:** lokalni smoke MORA da postavi `DJANGO_SECURE_SSL_REDIRECT=False` u smoke env-u (production.py mora ƒçitati `SECURE_SSL_REDIRECT` iz env-a ‚Äî vidi SM-D5/OQ-3-RESOLVED + Task 5.1) da bi HTTP/80 round-trip vratio realan 200 umesto 301‚ÜíHTTPS. Default ostaje `True` (prod hardening oƒçuvan). LOCAL-SMOKE koristi HTTP (port 80) ‚Äî HTTPS/443 cert je placeholder (SM-D5). Smoke je INFRA-VERIFY (curl/compose, kontejner se realno di≈æe ‚Äî Docker je dostupan), NE Playwright (needs_e2e=false).

4. **AC4 ‚Äî Nginx forwardu-je app-requeste na Django :8000; static servira Nginx direktno.** Non-static request (npr. `GET /sr/`) ‚Üí Nginx proxy ‚Üí Gunicorn :8000 ‚Üí Django response (Nginx je jedini izlo≈æen ulaz; Django/Gunicorn NEMA host port). `GET /static/<fajl>` ‚Üí Nginx servira iz `staticfiles` volume-a BEZ pogaƒëanja Django/Gunicorn-a (proxy access log NE pokazuje upstream hit za static). **‚öÝÔ∏è Volume je PRAZAN u 9.1** (`collectstatic` je deploy-step 9.2). ## Gotchas

- **G-1 (project-context Gotcha #5) ‚Äî LF line endings za `start.sh`.** Shell skripte u Docker kontejnerima MORAJU imati LF (NE CRLF) ‚Äî Windows host lako uvede CRLF ‚Üí `exec format error`/`bad interpreter` u Linux kontejneru. `compose/django/Dockerfile` komentar (linija 57) veƒá napominje LF za `entrypoint.sh`. `start.sh` isto. Proveri `.gitattributes` ILI editor LF lock.
- **G-2 (project-context Gotcha #22) ‚Äî Compose `$` interpolacija.** Docker Compose interpolira `$VAR` u `production.yml` i env vrednostima. `DJANGO_SECRET_KEY` koji sadr≈æi `$` (npr. iz `get_random_secret_key()`) se LOMI ‚Äî koristi `secrets.token_urlsafe(50)` (bez `$`) ILI escape `$$`. Isto za bilo koju env vrednost sa `$` u production.yml inline. (.env.example:19 veƒá upozorava.)
- **G-3 ‚Äî `proxy_set_header X-Forwarded-Proto $scheme` je OBAVEZAN (inaƒçe redirect loop).** production.py `SECURE_PROXY_SSL_HEADER=("HTTP_X_FORWARDED_PROTO","https")` + `SECURE_SSL_REDIRECT=True`: ako Nginx NE prosledi `X-Forwarded-Proto`, Django misli da je request HTTP ‚Üí 301 na HTTPS ‚Üí Nginx opet HTTP ka Django ‚Üí BESKONAƒåAN redirect loop. KLJUƒåNO za AC2. (Za lokalni HTTP smoke vidi SM-D5 ‚Äî SSL_REDIRECT se neutrali≈°e.)
- **G-4 ‚Äî prod `--no-dev` ne sme da razbije `just test` (dev put).** `compose/django/Dockerfile` linija 23 dev put (`--no-install-project`, ukljuƒçuje pytest/ruff) MORA ostati za `just test` (MP-D6). Prod `--no-dev` je ZASEBAN build target/arg ‚Äî NE prepisuj dev liniju. Lako pome≈°ati i razbiti lokalni test flow.
- **G-5 ‚Äî DB credentials u production.yml NIKAD hardkodovani `coric:coric` (kao local.yml:40).** local.yml ima `POSTGRES_PASSWORD: coric` INLINE (LOCAL ONLY komentar). production.yml MORA kroz `env_file`/Docker secrets (project-context:462 ‚ÄûSecrets: Hetzner Cloud + Docker secrets"). Kopiranje local.yml inline credentials u prod = security leak.
- **G-6 ‚Äî `migrate` NE u prod startup (project-context:478).** Lako kopirati `entrypoint.sh` (koji migrate-uje) u prod. Prod `start.sh` SAMO wait-for-db + gunicorn. Migrate = 9.2 deploy-step. (SM-D2.)
- **G-7 ‚Äî `add_header` u Nginx-u: koristi `always` flag.** Bez `always`, `add_header` se NE emituje na 4xx/5xx response-ima ‚Äî security headeri moraju biti i na error stranicama. `add_header X-Frame-Options "DENY" always;`.
- **G-8 ‚Äî Nginx static `alias` putanja mora da poklapa volume mount.** Ako django mount-uje `staticfiles` na `/app/staticfiles` i `STATIC_ROOT` je tamo, nginx `location /static/ { alias /app/staticfiles/; }` mora da mount-uje isti volume na istu putanju u nginx kontejneru. Mismatch ‚Üí 404 na static.
- **G-9 (step-02 Adversarial #2) ‚Äî `CSRF_TRUSTED_ORIGINS` MORA biti set u prod (inaƒçe 403 na svaki cross-origin POST).** Sa `CSRF_COOKIE_SECURE=True` + HTTPS preko reverse-proxy-ja, Django 4+ proverava Origin/Referer protiv `CSRF_TRUSTED_ORIGINS`. Ako je prazan ‚Üí 403 na admin-coric login, lead forme, sve POST-ove sa drugog scheme/host-a. `CSRF_TRUSTED_ORIGINS = env.list("DJANGO_CSRF_TRUSTED_ORIGINS", default=[])` (Task 5.1b). Vrednosti ukljuƒçuju scheme: `https://coricagrar.rs`. Smoke: `http://localhost`.
- **G-10 (step-02 Adversarial #1) ‚Äî `command:` override NE menja ENTRYPOINT (migrate i dalje radi).** Docker `command:` u compose-u override-uje SAMO Dockerfile CMD, NE ENTRYPOINT. Po≈°to Dockerfile ima `ENTRYPOINT ["/entrypoint.sh"]` (koji migrate-uje), `command: gunicorn ...` bi i dalje pokrenuo entrypoint.sh-migrate pa gunicorn kao arg. Prod MORA `entrypoint: ["/start.sh"]` (zameni ceo ENTRYPOINT) da migrate NE radi (project-context:478). start.sh kopiran u image na `/start.sh` (+ `chmod +x`).
- **G-11 (step-02 OQ-3/SM-D5) ‚Äî lokalni HTTP smoke daje 301 ako se SECURE_SSL_REDIRECT ne neutrali≈°e.** `SECURE_SSL_REDIRECT=True` + HTTP/80 (bez cert-a) ‚Üí Django 301 na HTTPS ‚Üí smoke nikad ne vidi 200. Smoke env MORA `DJANGO_SECURE_SSL_REDIRECT=False` (SM-D7 env-param). Prod default ostaje True. AC3: 200 je prolaz, 301 NIJE.
