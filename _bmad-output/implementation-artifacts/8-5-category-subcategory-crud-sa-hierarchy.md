# Story 8.5: Category + Subcategory CRUD sa Hierarchy

Status: review

## Opis

Kao **Marijana** (admin/Editor), želim **da kreiram i menjam kategorije (Traktori/Mehanizacija) i potkategorije sa hijerarhijom do 3 nivoa, sa per-lokal prevodima (sr/hu/en) i kontrolom redosleda prikaza**, da bih **mogla samostalno da organizujem taksonomiju mehanizacije tako da ima smisla za korisnike sajta, bez dev intervencije**.

Ovo je **PETA** Epic 8 story i DRUGA per-model CRUD admin (8.4 je isporučio Brand + Series inline; 8.5 nadograđuje `CategoryAdmin` i `SubcategoryAdmin`). Story **nadograđuje POSTOJEĆE `apps/brands/admin.py:CategoryAdmin` i `SubcategoryAdmin`** (trenutno minimalni stub-ovi: `SeoWarningAdminMixin + admin.ModelAdmin` sa `inlines=[SeoMetaInline]` iz 6.1 — vidi „Postojeće stanje" dole) u pun multi-locale hierarchy CRUD:

1. **Multi-locale tabovi (sr/hu/en)** za translatable polja (`name`, `description`) kroz `modeltranslation.admin.TranslationAdmin` — modeltranslation AUTO-renderuje per-locale polja, NE dodaje se ručno (mirror 8.4 / project-context:201).
2. **Hijerarhija do 3 nivoa Subcategory chain-a** — `Subcategory.parent` je self-FK (`"self"`, `CASCADE`, `blank/null`); `Subcategory.category` je FK ka `Category` (`CASCADE`). Stablo: `Category` → `Subcategory L1` → `Subcategory L2` → `Subcategory L3` (3 nivoa Subcategory chain-a, per `Subcategory.clean()` `_SUBCATEGORY_MAX_DEPTH=3`).
3. **Parent FK widget UX** — Editor bira `parent` (i `category`) kroz Django default FK select widget; `Subcategory.clean()` VEĆ enforce-uje (a) depth ≤ 3 i (b) cycle-guard (subcat.parent = subcat / petlja) → admin save MORA graceful-no propagirati te ValidationError poruke kao field/non-field greške (HTTP 200 re-render), NIKAD 500.
4. **Redosled prikaza (`display_order`)** — `Category` i `Subcategory` OBA imaju `display_order` (PositiveSmallIntegerField); `list_editable = ("display_order",)` na changelist-u omogućava prerasporedjivanje bez ulaska u svaki objekat (epics:1121 „order field omogućava prerasporedjivanje"; drag-and-drop = DEFER/OQ — bez novih dep-ova).
5. **`icon` polje** — Bootstrap Icons CSS klasa (CharField, blank); plain text input u formi (mirror 2-1 D8).
6. **Cascade-delete ponašanje** — brisanje `Category` CASCADE-uje vezane `Subcategory`-je; brisanje parent `Subcategory` CASCADE-uje child-ove. Django admin prikazuje standardnu **delete-confirmation stranu** sa listom related objekata pre potvrde (epics:1123 „cascade ili confirmation"). MEĐUTIM ako bilo koja Subcategory u cascade-u ima vezane `Product`-e (`Product.subcategory` = **PROTECT**), Django prikazuje „protected" stranu i blokira brisanje (NE 500) — vidi SM-D5 / G-4.
7. **Listing strane sa hijerarhijom** — `CategoryAdmin.list_display`: `name`, `is_for`, `display_order`, `slug`; `list_filter` na `is_for`. `SubcategoryAdmin.list_display`: `name`, `category`, `parent`, `display_order`, `slug` (parent kolona surfa-uje hijerarhiju); `list_filter` na `category` (i opciono `is_for` kroz `category__is_for`). `search_fields=("name_sr",)` (realna kolona — G-1).

**SCOPE GRANICA (KRITIČNO — 8.5 vs 8.4 vs 8.6):** 8.5 isporučuje SAMO **Category + Subcategory** (hierarchy CRUD). **Brand + Series** su VEĆ isporučeni u 8.4 (DONE) i 8.5 ih NE dira (`BrandAdmin`/`SeriesAdmin` ostaju netaknuti). **Product CRUD** (`apps/products/admin.py`, multi-locale, galerija/spec inline-ovi) je EKSPLICITNO Story 8.6 (epics:1125-1138) i VAN 8.5 scope-a. Vidi SM-D1.

## Kriterijumi prihvatanja

Iz epics.md:1112-1123 (Story 8.5), reconcilovano sa STVARNIM Category/Subcategory modelima (2-1) i postojećim 6.1 admin-om, i mirror-ujući 8.4 obrazac.

1. **AC1 — Multi-locale tabovi (sr/hu/en) za Category i Subcategory.** `CategoryAdmin` I `SubcategoryAdmin` su `modeltranslation.admin.TranslationAdmin` subklase (NE plain `admin.ModelAdmin`). Change/add forme prikazuju per-locale polja za `name`, `description` (`name_sr`/`name_hu`/`name_en`, `description_*`) koja modeltranslation AUTO-renderuje (NE ručno deklarisana; fieldsets koriste BAZNA imena — G-1). Forme se renderuju sa HTTP 200 za superuser-a I za Editor-a. **KRITIČNO (mirror 8.4 G-1):** `search_fields` MORA koristiti REALNU kolonu `name_sr` (NE virtuelni `name` → runtime `FieldError` na PRVI changelist search koji `manage.py check` NE hvata). Render-smoke test (changelist/add/change za oba modela) je OBAVEZAN (Task 7).

2. **AC2 — Hijerarhija do 3 nivoa (Subcategory chain) je editabilna i validirana.** Subcategory change/add forma sadrži `category` (FK select) i `parent` (self-FK select). Editor može da kreira `Category` → `L1` (parent=None) → `L2` (parent=L1) → `L3` (parent=L2). Pokušaj kreiranja **L4** (parent chain dubine 4) → admin save graceful odbijen sa `Subcategory.clean()` porukom „Subcategory hijerarhija ne sme prelaziti 3 nivoa dubine." (non-field / `__all__` ValidationError; admin re-renderuje formu, **HTTP 200, NIKAD 500**). NE re-implementirati depth-proveru u adminu — `Subcategory.clean()` je single-source (G-5/G-7).

3. **AC3 — Cycle-guard je graceful (testira se kroz change-form POST; dva puta sa RAZLIČITIM dostižnim invarijantama).** NE re-implementirati u adminu — `Subcategory.clean()` je izvor (G-5/G-7); cycle nastaje na CHANGE postojećeg objekta (za nov/unsaved `pk is None` → direktna samoreferenca u jednom save-u nije dostižna na add). **(a) Samoreferenca (A→parent=A):** postavljanje `Subcategory.parent` = sam taj Subcategory → admin save graceful ODBIJEN sa `Subcategory.clean()` porukom „Subcategory hijerarhija ne sme imati cikličnu referencu." (HTTP 200 re-render, NIKAD 500). Ovo je STVARNI cycle-rejection lock — `clean()` linija 380 (`current.pk == self.pk` eksplicitan pre-check) preživljava admin DB-reload put. **(b) Dvočvorna petlja (A parent=None, B parent=A, pa A→parent=B):** kroz admin reload put NIJE odbijena od strane model `clean()` — DB-reload prekida in-memory cycle-walk (`instance.parent=<svež B>` čiji `.parent` lazy-load-uje SVEŽ A sa `parent_id=None` jer petlja još nije perzistirana → `visited_ids` walk se završava, `full_clean()` PROLAZI, POST se SAČUVA 302). Zato je dostižna 8.5 invarijanta za 2-node-via-admin put SAMO „graceful — HTTP 200 re-render ILI 302 save, NIKAD 400/500". **Ovo je PRE-POSTOJEĆI model-validation jaz** (nasleđen iz 2-1, sibling OQ-5 cross-Category jaza i G-13 null-parent jaza), NE 8.5 posao da ga popravi (tražilo bi izmenu `Subcategory.clean()` → ruši 0-migracija AC11) — flag-ovan za buduću model-hardening story (vidi OQ-5).

4. **AC4 — `display_order` prerasporedjivanje (`list_editable`).** `CategoryAdmin.list_editable` I `SubcategoryAdmin.list_editable` uključuju `display_order` → Editor menja redosled direktno na changelist-u i čuva bulk (epics:1121). `list_display` MORA imati bar JEDNO ne-editabilno klikabilno polje kao link na change-view (npr. `name`) PRE `list_editable` polja (Django `list_editable` zahtev — inače `admin.E*`; G-9). Drag-and-drop reorder je DEFER (OQ-1 — traži dep/JS, van „0 dep").

5. **AC5 — `icon` (Bootstrap Icons klasa) editabilno.** `icon` polje je prisutno u change formi (plain text input — Bootstrap Icons CSS klasa, npr. `bi-tractor`); blank dozvoljen. NE uvoditi icon-picker widget (DEFER/OQ — van scope-a). Opciono `list_display` prikazuje `icon` kao tekst.

6. **AC6 — Cascade-delete confirmation + PROTECT-graceful.** Brisanje `Category` kroz admin → Django delete-confirmation strana prikazuje vezane `Subcategory`-je koji će biti CASCADE-obrisani (`Subcategory.category=CASCADE`); potvrda briše Category + sve njegove Subcategory-je. Brisanje parent `Subcategory` → CASCADE child-ove (`Subcategory.parent=CASCADE`). **MEĐUTIM:** ako bilo koja Subcategory (direktno ili kroz cascade) ima vezane `Product`-e (`Product.subcategory` = **PROTECT**, products/models.py:114-117), Django prikazuje **„protected" delete stranu** i blokira brisanje (HTTP 200, NE 500, NE tihi gubitak Product-a). Ovo je OČEKIVANO Django ponašanje — admin ga već daje; AC6 VERIFIKUJE (render-smoke) da delete-confirmation/protected strane vraćaju 200, NE da se implementira custom logika. NE menjati `on_delete` u modelu (0 migracija; SM-D5).

7. **AC7 — Listing strane sa hijerarhijom.** `CategoryAdmin.list_display` sadrži bar: `name` (link na change), `is_for`, `display_order`, `slug`; `list_filter=("is_for",)`. `SubcategoryAdmin.list_display` sadrži bar: `name` (link na change), `category`, `parent`, `display_order`, `slug` — `parent` kolona surfa-uje hijerarhiju (root Subcategory prikazuje prazno/`—` za parent); `list_filter` uključuje `category` (opciono `parent`). `parent`/`category` u list_display su FK → Django renderuje `__str__` (Subcategory.`__str__`=`name`, Category.`__str__`=`get_is_for_display() — name`); BEZ N+1 panike (changelist je admin, ne javni view) ali `list_select_related = ("category", "parent")` je preporučeno (G-10). `search_fields=("name_sr",)` na OBA (realna kolona — G-1). `ordering` prati model Meta (`Category.ordering=["is_for","display_order","name"]`, `Subcategory.ordering=["category","parent_id","display_order","name"]`).

8. **AC8 — `prepopulated_fields` slug + auto-gen reconciliacija.** OBA admina koriste `prepopulated_fields = {"slug": ("name",)}` (radi sa TranslationAdmin — mirror 8.4/PageAdmin G-2). `Category.save()`/`Subcategory.save()` ionako auto-generišu slug iz `name` ako je prazan. **Subcategory slug NIJE globally unique** (`UniqueConstraint(category, parent, slug)`) → dva Subcategory-ja sa istim `name` pod RAZLIČITIM (category, parent) imaju isti slug i to je VALIDNO; pod ISTIM (category, parent) → IntegrityError/ValidationError koju model full_clean/constraint hvata graceful (NE 500; testirati kroz admin POST — G-6). NE dodavati custom slug-uniqueness logiku u admin (model je izvor).

9. **AC9 — `view_on_site` hardening (Category I Subcategory NoReverseMatch guard).** **`Category.get_absolute_url`** reverse-uje `brands:category_traktori`/`category_mehanizacija` — ta URL imena **NISU registrovana** u `apps/brands/urls.py` (2-9/2-10 su koristili druga imena; `apps/search/views.py` VEĆ obmotava poziv `try/except NoReverseMatch` jer je potvrđeno da NE postoje) → CategoryAdmin „View on site" link bi bacio `NoReverseMatch` 500. Zato **`CategoryAdmin.view_on_site = False` je OBAVEZAN** (isto kao SubcategoryAdmin; mirror 8.4 SeriesAdmin G-3 / blog BL-5). Defanzivni guard — revidirati kad se ti javni URL-ovi REGISTRUJU u nekoj budućoj story (NE dodavati URL-ove ovde — van 8.5 admin-only / 0-migracija scope-a). **`Subcategory.get_absolute_url`** reverse-uje `brands:subcategory_listing_l{depth}` koji je registrovan SAMO za mehanizacija/prikljucna granu (2-11) → za Subcategory van te grane (ili nepotpun chain) takođe može baciti `NoReverseMatch` 500. Zato je **`SubcategoryAdmin.view_on_site = False` takođe OBAVEZAN** → i Category I Subcategory changelist/change ostaju 200, nikad ne padaju na NoReverseMatch.

10. **AC10 — RBAC: Editor I Superadmin CRUD-uju Category + Subcategory.** Category/Subcategory add/change/delete/changelist su dostupni i Editor-u (8.2 grupa VEĆ ima `brands.add/change/delete/view_category` + `..._subcategory` kroz `EDITOR_CONTENT_MODELS` — permissions.py:45-46) i Superadmin-u (`is_superuser`). Anoniman → admin login redirect (302). 8.5 NE menja `apps/accounts/permissions.py` niti re-grant-uje permisije (to je 8.2 posao; SM-D8, mirror 8.4 SM-D7). Editor add-view i change-view oba modela vraćaju 200 (forme se renderuju bez `admin.E*`).

11. **AC11 — Bez migracija, bez regresije.** 8.5 menja SAMO `apps/brands/admin.py` (`CategoryAdmin` + `SubcategoryAdmin` blokovi; eventualno mali admin forms helper) — `makemigrations brands --check` = „No changes" (0 šeme). Postojeći brands testovi (2-x katalog, 6-1 SeoMetaInline regression `test_seometa_admin_inline.py`, 8-4 Brand/Series CRUD testovi) OSTAJU zeleni. `BrandAdmin`/`SeriesAdmin` (8.4) NETAKNUTI. `/admin-coric/brands/category/` i `/admin-coric/brands/subcategory/` changelist + add + change = 200 za staff. `python manage.py check` = 0 `admin.E*`.

## Gotchas

- **G-1 — `search_fields` na VIRTUELNOM translatable polju → `FieldError` na changelist search (runtime, NE `manage.py check`).** Registrovan modeltranslation `name` je virtuelno polje; `search_fields=("name",)` baca `FieldError` na PRVI changelist search query — `manage.py check` ga NE hvata. Koristi REALNU kolonu `search_fields=("name_sr",)` (mirror 8.4 G-1 / blog / pages). `list_display=("name",...)` je OK (atribut/`__str__`). `prepopulated_fields={"slug":("name",)}` radi (modeltranslation patch — verifikovano 8.4).
- **G-2 — MRO red mixina: `SeoWarningAdminMixin` PRVI, pa `TranslationAdmin`.** `class CategoryAdmin(SeoWarningAdminMixin, TranslationAdmin)` (NE obrnuto) — `SeoWarningAdminMixin.save_formset` poziva `super().save_formset()` koji mora ići u TranslationAdmin lanac (mirror 8.4 BrandAdmin / blog.PostAdmin). Isto za SubcategoryAdmin.
- **G-3 — OBA `get_absolute_url`-a (Category I Subcategory) reverse-uju NEREGISTROVANA URL imena → `view_on_site=False` na OBA admina OBAVEZAN.** `Category.get_absolute_url` reverse-uje `brands:category_traktori`/`category_mehanizacija` koja **NE postoje** u `apps/brands/urls.py` (potvrđeno: `apps/search/views.py` VEĆ guard-uje taj poziv `try/except NoReverseMatch` jer imena ne postoje). `Subcategory.get_absolute_url` reverse-uje `brands:subcategory_listing_l{depth}` registrovan SAMO za mehanizacija/prikljucna granu (2-11) → van te grane isto puca. Bez `view_on_site=False`, „View on site" link na change-view OBA modela → `NoReverseMatch` 500. REŠENJE: **`CategoryAdmin.view_on_site = False` I `SubcategoryAdmin.view_on_site = False`** (OBAVEZNO — AC9/Task 2/Task 3; mirror 8.4 SeriesAdmin precedent). Defanzivni guard — revidirati kad se javni URL-ovi registruju (NE dodavati ih u 8.5).
- **G-4 — Subcategory delete sa PROTECTed Product = „protected" strana (NE 500, NE tihi cascade).** `Product.subcategory=PROTECT` (products/models.py:114-117). Brisanje Subcategory (direktno ILI kroz Category-cascade) koja ima Product-e → Django prikazuje „cannot delete, protected related objects" stranu (200). OČEKIVANO — Editor prvo mora premestiti/obrisati Product-e. NE pisati custom delete; NE 500. Verifikuj render (Task 5).
- **G-5 — NE re-implementirati depth/cycle u adminu — `Subcategory.clean()` je single-source.** `Subcategory.clean()` (models.py:365-396) enforce-uje depth-3 + cycle kroz `full_clean()` (poziva ga admin ModelForm). Re-implementacija u admin formi = drift + dva izvora istine. Admin samo propušta ValidationError kao non-field form grešku (200 re-render). Isto za sve model-nivo validacije (G-7).
- **G-6 — Subcategory slug NIJE globally unique — `UniqueConstraint(category, parent, slug)`.** Za razliku od Category (`slug unique=True` globally), Subcategory slug je unique SAMO unutar (category, parent). Dva Subcategory ista `name`+ISTI scope → constraint violation koju full_clean/IntegrityError hvata graceful (200 form-greška, NE 500). NE dodavati custom slug-uniqueness u admin. Testirati kroz admin POST (dupli slug u istom scope-u → graceful odbijen).
- **G-7 — NE duplirati model validaciju u adminu.** `Category.clean()`/`Subcategory.clean()` (+ constraints) već validiraju depth/cycle/slug-scope. Admin NE re-implementira — propušta model ValidationError. (Mirror 8.4 G-7.)
- **G-8 — 6.1 SeoMetaInline regression.** `apps/seo/tests/test_seometa_admin_inline.py` lock-uje da receiving admini (uklj. Category, Subcategory) zadrže SeoMetaInline bez `admin.E*`. Konverzija u TranslationAdmin MORA zadržati `SeoMetaInline` u `inlines` oba. Pokreni taj test (Task 7).
- **G-9 — `list_editable` + `list_display_links` zamka (`admin.E*`).** `list_editable=("display_order",)` zahteva: (a) `display_order` JESTE u `list_display`; (b) bar JEDNO polje u `list_display` je klikabilan link (`list_display_links`) i NIJE u `list_editable` (inače `admin.E124/E125`). REŠENJE: `name` ostaje prvo polje u `list_display` (default link) i NIJE u `list_editable`. `manage.py check` MORA biti 0 `admin.E*` (Task 7).
- **G-10 — `list_display` FK kolone (`category`/`parent`) → N+1 na changelist render.** SubcategoryAdmin `list_display` ima `category` i `parent` FK → svaki red triggeruje query za `__str__` ako nema `list_select_related`. REŠENJE: `list_select_related = ("category", "parent")` na SubcategoryAdmin. Admin-only (ne javni view) ali jeftin guard; opciono assertNumQueries lock. Category `__str__` poziva `get_is_for_display()` (NE query — choices su in-memory) pa CategoryAdmin nema isti problem.
- **G-11 — `name_sr`/`is_for` bezuslovno obavezni (modeltranslation default-locale + blank=False).** `Category.name`/`Subcategory.name` blank=False + sr default → `name_sr` promovisan u required (mirror 8.4 G-11); `Category.is_for` blank=False bez default → required. NE relaksirati. Prazan → graceful 200 form-greška, nikad full_clean escape. `slug` je required na admin form-u (`prepopulated_fields` JS puni iz `name` u browseru; no-JS test scraper mora eksplicitno dati `slug`). Admin-POST testovi vrte realan `name_sr`/`is_for` required.
- **G-12 — Ako se uvodi custom ModelForm: `Meta.fields` ILI `Meta.exclude` OBAVEZAN.** (Mirror 8.4 G-12.) Django ModelForm bez `fields`/`exclude` → `ImproperlyConfigured` na import (sajt 500). Ako se uvodi (npr. za parent-scoping) — poštuj ga.
- **G-13 — `UniqueConstraint(category, parent, slug)` NE hvata duplikat slug-a za null-parent (L1 root) na PostgreSQL.** `Subcategory` constraint je `(category, parent, slug)`; na PostgreSQL NULL je DISTINCT (default SQL semantika) → dva reda sa `(category=X, parent=NULL, isti slug)` NE krše constraint → dve L1 (root) Subcategory pod ISTOM Category mogu deliti slug (potencijalno dvosmislen `get_absolute_url`/listing). Za NON-NULL parent constraint radi normalno (hvata duplikat → graceful 200). Ovo je PRE-POSTOJEĆE model ograničenje (nasleđeno iz 2-1; UniqueConstraint NEMA `nulls_distinct=False`) — **NIJE 8.5 posao da menja** (tražilo bi migraciju → ruši 0-migracija AC11). Dokumentovano kao prihvatljiv v1 jaz / flag; popravka (`condition`-bazirani partial constraint za parent IS NULL, ili `nulls_distinct=False` na PG≥15) je buduća migracija-story. Testiranje: vidi AC8 Testing note (oba slučaja).
