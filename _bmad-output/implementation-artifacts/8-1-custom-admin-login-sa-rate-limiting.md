# Story 8.1: Custom Admin Login sa Rate Limiting

Status: review

## Opis

As a **admin**,

I want **siguran login URL koji nije `/admin/` (veƒá `/admin-coric/`), sa brute-force za≈°titom (django-axes lockout posle 5 neuspelih poku≈°aja u 15 min ‚Üí 1h lock), email-as-username login formom i 4h session timeout-om**,

so that **bot-ovi ne mogu trivijalno da probaju credential-e na predvidivom `/admin/` URL-u, a ponavljani neuspeli login-i se automatski blokiraju**.

Ovo je **PRVA story Epic 8 (Admin Dashboard & Content Management)** ‚Üí prebacuje **epic-8 backlog‚Üíin-progress**. ƒåisto **FOUNDATIONAL SECURITY HARDENING** admin login-a. Sve naredne Epic 8 stories (8.2 RBAC, 8.3 dashboard, 8.4..8.9 CRUD admini) se oslanjaju na ovaj za≈°tiƒáen admin entry-point.

Rezultat: `/admin-coric/` (bez locale prefiksa), a `/admin/` i `/sr/admin/` vraƒáaju 404.

   - `"axes.middleware.AxesMiddleware"` kao **POSLEDNJI** u `MIDDLEWARE` (POSLE Authentication middleware-a; axes docs).
   - `AXES_FAILURE_LIMIT = 5` (**OFF-BY-ONE SEMANTIKA ‚Äî SM-D16:** axes zakljuƒçava kada broj neuspelih poku≈°aja DOSTIGNE `AXES_FAILURE_LIMIT`, tj. **5. neuspeli poku≈°aj SAM TRIGGERUJE lockout**, ne 6. Task 1 i dalje verifikuje protiv instalirane axes verzije, ali TEA koristi ovu konkretnu vrednost za settings-assertion test (AC11/AC5).
